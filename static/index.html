Help & Glossary

What is TokenOS?

A deterministic execution kernel for LLM agents. Routing, verification, loop detection and provider choice happen in code, with zero tokens — the model is only invoked for work that truly requires generation. The headline metric is cost per successful task.

Route types

REUSEA verified or statically-checked answer for this exact goal already exists — served from cache for zero tokens.
DIRECTTiny, unambiguous task — answered with a minimal prompt on the cheapest provider.
PATCHSmall, well-scoped edit to existing material.
IMPLEMENTReal generation work — full pipeline with verification.
PARTIALResume an interrupted task from its saved state.
DELEGATEBig enough to hand to a sub-agent with a compressed delegation packet.
ASKThe task is too ambiguous — the cheapest action is asking a clarifying question.
ESCALATERepeated failures or loops detected — a human should look.

Glossary

Dry-run modeA fault-injectable mock provider exercises the full pipeline offline. No API key, no spend — perfect for evaluation.
Shadow pricingEvery candidate provider is priced in code before any call; the cheapest viable one goes first.
UCB1 banditA learning algorithm that balances trying providers (exploration) with using the proven best one (exploitation).
Drift watchdogCompares billed token counts to local estimates; flags providers whose cost predictions are off.
Solution cacheOutputs statically-checked or verified by tests, keyed by goal + constraints; an identical request is replayed for zero tokens.
Flight recorderA per-task trace of decisions, prompts, responses, rescues and verification — open any task to inspect it.
